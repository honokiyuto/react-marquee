### **1\. `<marquee>`タグの基本的な機能と属性**

`<marquee>`（マーキー）タグは、ウェブページ上でテキストや画像を電光掲示板のように自動でスクロールさせるためのHTML要素です。その動作は、さまざまな属性を指定することで細かく制御することが可能です。

[marquee: マーキー要素 - HTML | MDN](https://developer.mozilla.org/ja/docs/Web/HTML/Reference/Elements/marquee)

---

### **2\. `<marquee>`タグの歴史と物語**

#### **2.1. 誕生の背景：ブラウザ戦争の産物**

`<marquee>`タグは、**1995年に公開されたInternet Explorer 2.0で初めて実装された、Microsoft独自の機能**でした 。

両社はウェブ標準の策定を待たずに独自のHTMLタグ（独自拡張機能）を次々と実装し、自社ブラウザの優位性を示そうと競い合っていました 。

開発の意図は、劇場やニュース速報で使われる電光掲示板のように、ウェブページ上で情報を目立たせ、手軽に動的な表現を実現することでした 。

<div>
  <marquee>
    これはマーキー要素です
  </marquee>
</div>

#### **2.2. 日本における独自の普及と「ガラケー」文化**

`<marquee>`タグは、2000年代初頭の日本で特に爆発的に普及しました。その背景には、当時の技術的制約と独自の携帯電話文化がありました。

- **技術的制約**:
  - **狭い画面**: 全角10文字程度しか表示できないフィーチャーフォン（ガラケー）の画面では、長い文章を改行せずに表示させる実用的な手段でした 。
  - **乏しい表現力**: CSSやJavaScriptのサポートが不十分またはキャリアごとに異なっていたガラケーにおいて、動きをつけられる数少ない貴重な手法でした 。

- **文化的背景**:
  - **「ホムペ」と「デコる」文化**: 中高生を中心に個人のホームページ（ホムペ）作成が流行し、サイトを目立たせるための「デコレーション」として`<marquee>`は最適でした 。
  - **お知らせ機能**: サイトのトップで歓迎メッセージや更新情報を流す、電光掲示板のような役割も果たしました 。

#### **2.3. 非推奨への道：なぜ`<marquee>`は嫌われたのか**

その手軽さから広く使われた一方で、`<marquee>`タグは多くの問題点を抱えており、ウェブコミュニティや専門家から厳しい批判を受けました。

##### **技術的・標準化の問題**

- **非標準**: IEの独自機能であり、ウェブ技術の標準を定めるW3C（World Wide Web Consortium）の公式仕様に一度も採用されませんでした 。
- **関心の分離の原則への違反**: 見た目（プレゼンテーション）を制御する機能が、文書構造を定義すべきHTMLに混在しており、HTMLとCSSの役割を分離するという現代のウェブ開発の基本原則に反していました 。
- **HTML5での廃止**: これらの理由から、HTML5の仕様策定時に正式に「廃止された機能（obsolete）」と位置づけられ、使用しないことが強く推奨されるようになりました 。

##### **ユーザビリティとアクセシビリティの問題**

ユーザビリティの第一人者であるヤコブ・ニールセン氏をはじめとする専門家たちは、早くからこのタグがユーザー体験に与える悪影響に警鐘を鳴らしていました。

- **可読性の著しい低下**: 動いているテキストは静止しているテキストより格段に読みづらく、ユーザーに不必要な認知的負荷をかけます 。
- **ユーザーのコントロールの阻害**: ユーザーが動きを止めたり、速度を調整したりできず、ウェブサイトの閲覧における主導権を奪います。これはニールセン氏が提唱するユーザビリティ原則にも反するものでした 。
- **注意散漫の原因**: 本来集中すべき主要なコンテンツからユーザーの注意を逸らしてしまいます 。
- **プロフェッショナリズムの欠如**: 過剰な装飾はサイトを素人っぽく、信頼性の低いものに見せると指摘されました 。
- **アクセシビリティ障壁**: スクリーンリーダーなどの支援技術が内容を正しく読み上げられない可能性があり、特定のユーザーを情報から排除する結果につながりました 。

これらの批判は、ウェブデザインが単なる「見た目の目新しさ」から「ユーザーにとっての使いやすさ」を重視する「ユーザー中心設計」へと移行していく大きなきっかけの一つとなりました。

### **3\. `<marquee>`タグのその後と現代**

#### **3.1. CSSによる代替案の模索と挫折**

`<marquee>`の機能をCSSで代替しようという動きもかつて存在しました。「CSS Marquee Module Level 3」として、`marquee-style`や`marquee-direction`といったプロパティが提案され、一時はW3Cの草案（Working Draft）にもなりました 。

しかし、この提案は最終的に規格化されませんでした。その理由は以下の通りです 。

1. **ブラウザの実装が進まなかった**: 主要なブラウザベンダーによる十分な実装（採用）がありませんでした 。
2. **代替技術の存在**: より汎用的で強力な**CSS Animations**で同様の効果が実現可能であり、専用のプロパティは不要と判断されました 。
3. **アクセシビリティへの懸念**: 動きのあるコンテンツがもたらす可読性や制御不能といった問題が、新しい標準として採用する上での障壁となりました 。

#### **3.2. 現代における再評価と意図的な活用**

非推奨となった`<marquee>`ですが、その「悪名」や歴史的背景を逆手にとり、特定の文脈で意図的に使われることがあります。

- **ノスタルジアとアート**: 1990年代のジオシティーズ（GeoCities）に代表されるような、混沌として自由だった初期インターネットへのノスタルジアを表現する手段として、アートプロジェクト（例: Cameron's World）や個人サイト（例: Neocities）で活用されています 。
- **ブルータリスト・ウェブデザイン**: 現代の洗練され均質化したデザインへの批評やアンチテーゼとして、あえて「醜く」「荒削り」な表現を用いるブルータリズムの文脈で、その無骨な動きが再評価されています 。
- **ギーク・ハッカー文化**: 古い技術をあえて使いこなし、現代のブラウザで意図した挙動をさせること自体に面白さや「ハックバリュー」を見出すハッカー文化の文脈でも用いられます 。

#### **3.3. モダンな代替実装と活用事例**

現在、`<marquee>`と同様の効果はCSSアニメーションで実装するのが標準です。これにより、アクセシビリティにも配慮した、より洗練された実装が可能になります 。

**簡単なCSS実装例:**

```html
<div class="marquee-container">
  <p class="marquee-text">
    これはCSSアニメーションによるスクロールテキストです。
  </p>
</div>
```

```css
.marquee-container {
  width: 100%;
  overflow: hidden; /* はみ出した部分を隠す */
  box-sizing: border-box;
}
.marquee-text {
  display: inline-block;
  white-space: nowrap; /* テキストを折り返さない */
  padding-left: 100%; /* 右端からスタートさせる */
  animation: marquee-anim 15s linear infinite;
}
@keyframes marquee-anim {
  0% {
    transform: translateX(0);
  }
  100% {
    transform: translateX(-100%); /* 左端へ完全に移動させる */
  }
}
/* ユーザーが動きの抑制を要求した場合にアニメーションを停止 */
@media (prefers-reduced-motion: reduce) {
  .marquee-text {
    animation: none;
  }
}
```

この効果は、現代のウェブサイトでもデザインのアクセントとして効果的に活用されています。

- **NUTION**: パーソルキャリアのデザイン組織のサイトでは、組織が取り組む領域やカルチャーを象徴するキーワードをダイナミックに見せるためにスクロール効果が使われています 。
- **靴下屋 2022 SPRING COLLECTION**: 特設サイトの導入部分で、遊び心のあるタイポグラフィを流し、ユーザーをコレクションの世界観に引き込んでいます 。

これらの事例では、情報を詰め込むためではなく、ブランドイメージの強調や視覚的な面白さを加える装飾的な目的で、控えめかつ洗練された形で利用されています 。

#### **3.4. 将来のサポート状況**

2025年現在、Chrome、Firefox、Safariなどの主要ブラウザは、後方互換性のために`<marquee>`タグの表示をサポートし続けています 。

しかし、MDN Web Docsには「この機能はいつでも動作しなくなる可能性がある」と明記されており、将来的なサポートは保証されていません 。したがって、新規での使用は絶対に避けるべきです。

### **まとめ**

`<marquee>`タグは、1990年代のブラウザ戦争の最中にIEの独自機能として生まれ、その手軽さから、特に技術的制約の多かった日本のガラケーサイトや個人のホームページで爆発的に普及しました。しかし、ウェブ標準からの逸脱や、ユーザビリティ・アクセシビリティ上の多くの問題点から専門家たちに厳しく批判され、HTML5で正式に廃止されるという運命を辿りました。

その物語はそこで終わらず、CSS3での標準化が一度は検討されたものの、より汎用的なCSSアニメーションにその座を譲りました。現代では、その「悪名」やノスタルジックな雰囲気から、アートや批評的なデザインの文脈で意図的に再利用されることがあります。そして、その視覚効果はCSSアニメーションという洗練された形で受け継がれ、モダンなウェブデザインのアクセントとして新たな命を吹き込まれています。

`<marquee>`タグの歴史は、ウェブ技術の進化、標準化の重要性、そしてユーザー中心設計への思想の変遷を映し出す、非常に興味深い物語と言えるでしょう。
